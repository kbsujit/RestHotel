RESTful Web Service API

Go Back to Home Page

REST is an acronym for Representational State Transfer (see: http://en.wikipedia.org/wiki/REST. This means that all interaction from client to server is via normal HTTP methods: GET, POST, PUT, PATCH, DELETE and URIs (Universal Resource Identifiers) that represent the data access methodology. All data payloads to and from the server are in JSON format. In the API details below, replace "localhost:8080" with the actual server and port used.

Retrieve All Hotel Records
HTTP Method: GET
URI: http://localhost:8080/hotels
Response Data Type: application/json

Create New Hotel Record
HTTP Method: POST
URI: http://localhost:8080/hotels
Request Payload Content Type: application/json 
     jgl-mac: curl -i -X POST -H "Content-Type:application/json" -d '{  "address" : "1234 Street",  "city" : "Chicago", "name" : "Best Western", "zip" : "60612" }' http://localhost:8080/hotels

        HTTP/1.1 201 Created
        Server: Apache-Coyote/1.1
        Location: http://localhost:8080/hotels/12
        Content-Length: 0
        Date: Mon, 28 Apr 2014 14:52:37 GMT


Update Hotel Record By ID
HTTP Method: PATCH
URI: http://localhost:8080/hotels/12
Request Payload Content Type: application/json
Response Payload Data Type: html (nothing actually comes back) 
     jgl-mac: curl -i -X PATCH -H "Content-Type:application/json" -d '{  "address" : "444 Street",  "city" : "Chicago", "name" : "Best Western", "zip" : "60612" }' http://localhost:8080/hotels/12

        HTTP/1.1 204 No Content
        Server: Apache-Coyote/1.1
        Date: Mon, 28 Apr 2014 14:56:44 GMT


Delete Hotel Record By ID
HTTP Method: DELETE
URI: http://localhost:8080/hotels/12
Response Payload Data Type: html (nothing actually comes back) 
     jgl-mac: curl -X DELETE http://localhost:8080/hotels/12

     (no response)
